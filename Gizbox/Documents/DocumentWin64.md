# 在Windows64平台的栈帧结构    

```  
高地址
+------------------+
| 参数5, 6, 7...    | <- 超出4个参数的部分
+------------------+
| Shadow Space     | <- 32字节（调用者分配）
| (4个8字节槽位)    |
+------------------+
| 返回地址          | <- call指令推入
+------------------+
| 保存的RBP         | <- 被调用者保存
+------------------+
| 保存的其他非易失性 | <- 被调用者保存
| 寄存器            |
+------------------+
| 局部变量          | <- 编译器分配
+------------------+
| 寄存器溢出槽位     | <- 编译器管理
+------------------+
| 对齐填充          | <- 保持16字节对齐
+------------------+ <- RSP指向这里
低地址
```


# 结构体（Struct）底层机制（Win64 后端）

### 内存布局

- 结构体在语义分析阶段计算字段偏移、对齐和总大小。  
- 结构体类型会带大小信息（内部形如`(struct:size)Name`）。  
- 字段访问按固定偏移生成，不走虚函数表。  

### 传参规则（按大小分两类）

- **小结构体（`size <= 8`）**：按值传递。  
  - 前 4 个 ABI 参数槽优先走寄存器（`RCX/RDX/R8/R9`），同时写入调用栈参数槽。  
  - 超过 4 个参数走调用栈参数槽。  
- **大结构体（`size > 8`）**：按地址传递（隐式 by-ref）。  
  - 调用者先在当前调用栈帧的临时区拷贝一份结构体实参。  
  - 把这份临时拷贝的地址作为实参传递（寄存器/栈槽）。  

### 返回规则（按大小分两类）

- **小结构体（`size <= 8`）**：返回值放在`RAX`。  
- **大结构体（`size > 8`）**：使用隐藏返回指针参数。  
  - 调用者在调用前分配返回缓冲区，并把其地址作为“隐式第 0 参数”传入。  
  - 被调函数将返回结构体内容拷贝到该地址。  
  - 因为插入了隐藏参数，用户可见参数的 ABI 索引整体后移 1。  

### 调用栈位置（本实现）

以一次调用点为例，在`sub rsp, rspSub`后：

- `[rsp + 8*i]`：第`i`个 ABI 参数槽（含 shadow space 槽位和超过寄存器数的栈参数槽）。  
- 大结构体实参临时拷贝区：位于参数槽之后的临时区。  
- 大结构体返回缓冲区：同样位于临时区（由调用者分配）。  

在被调函数内：

- `[rbp+16]`是参数槽 0（若大结构体返回则这里是隐藏返回指针）。  
- 其后参数槽按 8 字节递增。  

### 对齐规则

- 调用前总的`rspSub`会统一补齐到 16 字节对齐（满足 Win64 `call` 前对齐要求）。  
- 大结构体临时区与隐藏返回缓冲区按 8 字节向上对齐分配。  

### 生命周期与清理

- 大结构体实参临时拷贝区、隐藏返回缓冲区都在调用者栈上。  
- 调用结束后通过`add rsp, rspSub`一次性回收，不需要额外“释放”动作。  
- 被调函数不负责清理这些调用点临时区。  

### 当前优化状态

- 目前**没有**RVO/NRVO优化。  
- 结构体返回仍以显式拷贝路径实现（小结构体寄存器返回 / 大结构体隐藏返回缓冲区拷贝）。
